Robson · v0.104 · Execution & risk engine

An execution runtime for leveraged markets.

Robson is not a trading bot. It manages what happens after a decision is made — order lifecycle, risk invariants, and failure handling — with deterministic semantics and a full audit trail.

Primary concerns · three subsystems
01 — EXECUTION
Plan. Validate. Execute.
A strict PLAN → VALIDATE → EXECUTE pipeline. No unvalidated order reaches the exchange. Dry-run is the default; live execution requires explicit acknowledgement.
02 — RISK
Enforced, not advised.
Position sizing, stop logic, and liquidation distance are invariants of the runtime. Every exit carries a typed reason code — stop hit, risk limit, manual, validation failure, timeout.
03 — AUDIT
Append-only event trail.
State transitions, risk checks, and external flows are recorded with causality links. The lifecycle of any position is reconstructible at any point in time.
CLI · deterministic

Dry-run by default. Live by declaration.

The execution pipeline defaults to simulation. Live execution requires --live --acknowledge-risk and a prior passing validation.

bash · robson09:14:28
# 1. create a plan
robson plan buy BTCUSDT 0.001 --limit 67420
  → plan-2847 created · pending validation

# 2. validate against risk invariants
robson validate plan-2847 --client-id 1 --strategy-id 5
  → 4/5 checks passed · stop distance below floor

# 3. execute (dry-run)
robson execute plan-2847 --client-id 1
  → simulated · no order sent

# 4. execute (live)
robson execute plan-2847 --live --acknowledge-risk
In leveraged markets, how you execute matters more than what you execute. A sound signal with poor execution, missing stop logic, or uncontrolled position sizing will lose capital.
